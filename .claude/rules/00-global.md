# 00 — 전역 규칙 & 작업 프로세스

## 작업 프로세스 (매 작업 필수)

```
1. 관련 파일 Read → 근본 원인 분석
2. 분석 결과 보고 (무엇이, 왜 깨졌는지)
3. 대표님 승인
4. 구현 → tsc --noEmit 통과 확인
5. 커밋
```

## 전역 금지사항

- ❌ 근본 원인 없이 패치/우회로 때우기
- ❌ 승인 없이 Phase 2 기능 추가
- ❌ `tsc --noEmit` 실패 상태로 커밋
- ❌ 소스(`C:\Users\장우경\oomni`) 외 경로 수정
- ❌ 삭제 대상 파일(code-audit.md ❌DELETE) 수정
- ❌ 기존 동작하는 코드 "개선"이라는 이유로 리팩토링 (요청 없을 시)

## 버전 관리 규칙

- 버그 수정: x.x.**X** (patch)
- 기능 추가: x.**X**.0 (minor)
- 브레이킹 체인지: **X**.0.0 (major)

## 커밋 후 릴리즈 순서

```
git push → gh release create → docs/index.html 버전 업데이트 → push
```
