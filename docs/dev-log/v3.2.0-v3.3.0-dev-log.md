## v3.3.0 변경 사항

### 배경 — 왜 Pencil을 교체했나?

Pencil MCP는 자동 기동 및 MCP 연결은 성공하지만,
**디자인 결과물이 OOMNI 중앙 패널 안에서 직접 보이지 않는** 한계가 있었다.

Pencil은 독립 앱으로 실행되며 iframe/BrowserView로 내장 불가 → PNG export를 3초 폴링으로 표시하는 구조였으나 UX가 열악했다.

Claude Design (Anthropic Research Preview) 분석 결과, `claude-opus-4-7` 모델이 Claude Design 수준의 완성도 높은 단일 HTML 파일을 생성할 수 있음을 확인.
→ 기존 `DesignCenterPanel`의 `extractHtml()` + `<iframe srcdoc>` 실시간 렌더링 구조를 그대로 활용 가능.

### TASK-03: Pencil 기술부채 전체 제거

#### ptyService.ts
**파일**: `backend/src/services/ptyService.ts`

제거된 코드 (design role 전용 Pencil 자동 기동):
```typescript
// 제거됨
if (role === 'design') {
  setTimeout(() => {
    ptyProcess.write(
      `$p = Get-Process "Pencil" -ErrorAction SilentlyContinue; ` +
      `if (-not $p) { Start-Process "$env:LOCALAPPDATA\\Programs\\Pencil\\Pencil.exe"; Start-Sleep -Seconds 4 }; ` +
      `node "${cliPath}" --dangerously-skip-permissions\r`
    );
  }, 500);
}
```

변경 후: 모든 역할 동일하게 `node "cliPath" --dangerously-skip-permissions` 실행

**유지**: PowerShell wrapper (`-NoLogo -NoExit`) — ConPTY exit code 1 문제 해결 핵심, 제거하면 안 됨

#### agents.ts
**파일**: `backend/src/api/routes/agents.ts`

제거된 엔드포인트:
- `GET /api/agents/:id/preview-image` — workspace PNG 서빙 (Pencil export_nodes 결과)
- `GET /api/agents/:id/pencil-status` — 로컬 Pencil MCP 바이너리 존재 확인
- `import * as os from 'os'` — 위 두 엔드포인트에서만 사용, 제거 후 TS6133 오류 방지

## v3.2.0 변경 사항 (이전 세션)

### 1. 터미널 exit code 1 완전 수정

**파일**: `backend/src/services/ptyService.ts`

**근본 원인**: Claude Code CLI TUI가 Electron 바이너리와 ConPTY 루트로 spawn 되면 TUI 초기화 실패 (exit code 1)

**수정**: `powershell.exe -NoLogo -NoExit`를 PTY 루트로 spawn → 500ms 후 `node "cliPath" --dangerously-skip-permissions` 자동 입력

모든 역할(design/build/ops/research/content/ceo)에 동일 방식 적용.

## 전체 봇 파이프라인 (현재 기준)

```
Research Bot
  → Design Bot (UI/UX 방향)
  → Content Bot (리서치 → 블로그 포스팅)
  → CEO Bot (시장 분석 브리핑)

Design Bot (claude-opus-4-7 HTML 스트리밍)
  → Build Bot [프론트엔드] (HTML 목업 → React 구현)
  → Content Bot (디자인 결과물 소개)

Build Bot [프론트엔드]
  → Build Bot [백엔드] (API 엔드포인트 목록 전달)

Build Bot [백엔드]
  → Ops Bot [인프라] (배포 요구사항, 환경변수)

Ops Bot
  → Content Bot (배포된 서비스 기능 소개)
  → CEO Bot (운영 현황 보고)

Content Bot
  → CEO Bot (콘텐츠 성과 분석)
```
