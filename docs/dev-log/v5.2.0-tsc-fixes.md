## 백엔드 수정 (8개 에러)

### 1. `src/services/cdpTriggerService.ts`
- **에러**: TS6133 — `db` 파라미터 선언됐지만 사용 안 됨 (line 151)
- **수정**: `db: DbClient` → `_db: DbClient`

### 2. `src/services/codexService.ts`
- **에러**: TS2307 — `'openai'` 모듈을 찾을 수 없음 (line 28)
- **원인**: `openai` 패키지가 선택적 의존성(미설치)이지만 dynamic import에서도 TS가 타입 확인
- **수정**: `// @ts-expect-error openai is optional peer dependency` 주석 추가

### 3. `src/services/growthIngestionService.ts`
- **에러**: TS2307 — `'./config'` 모듈을 찾을 수 없음 (line 10)
- **원인**: `services/` 하위에서 같은 디렉토리(`./config`)로 import했으나 `config.ts`는 `src/`에 위치
- **수정**: `'./config'` → `'../config'`

### 4. `src/services/leadScoringService.ts`
- **에러**: TS6133 — `profileFilter` 선언됐지만 사용 안 됨 (line 78)
- **원인**: 쿼리 문자열에 인라인 조건식을 사용하고 있어 `profileFilter` 변수가 dead code
- **수정**: `profileFilter` 선언 라인 삭제

### 5. `src/services/roleExecutors/growth.ts` (3개 에러)
- **에러 1** TS6133: `uuidv4` import 사용 안 됨 (line 17) → import 라인 삭제
- **에러 2** TS2307: `'../logger'` 경로 오류 (line 23)
  - `roleExecutors/` 하위에서 `src/logger.ts`로 가려면 `../../logger`
  - `'../logger'` → `'../../logger'`
- **에러 3** TS2322: `ProductInfo` 타입을 `Record<string, unknown>`에 할당 불가 (line 58)
  - `ProductInfo`에 index signature 없음
  - **수정**: `await ingestUrl(inputUrl) as unknown as Record<string, unknown>` 타입 캐스트

### 6. `src/services/snsPublisherService.ts`
- **에러**: TS6133 — `accessToken` 파라미터 사용 안 됨 (line 312)
- **수정**: `accessToken: string` → `_accessToken: string`

## 프론트엔드 수정 (39개 에러)

### 패턴 A — 불필요한 `React` import 제거 (7개 파일)

`jsx: react-jsx` 설정 시 JSX 변환에 React 네임스페이스 불필요

| 파일 | 수정 내용 |
|---|---|
| `components/board/CalendarView.tsx` | `import React, { useState }` → `import { useState }` |
| `components/board/ListView.tsx` | `import React from 'react'` → 라인 삭제 |
| `components/board/QuickAddTask.tsx` | `import React, { useState, useRef }` → `import { useState, useRef }` |
| `components/board/TaskDetail.tsx` | `import React, { useState, useEffect }` → `import { useState, useEffect }` |
| `pages/DesignStudio.tsx` | `import React, { useState }` → `import { useState }` |
| `pages/MissionBoard.tsx` | `import React, { useEffect, ... }` → `import { useEffect, ... }` |
| `pages/OpsCenter.tsx` | `import React, { useState, ... }` → `import { useState, ... }` |

### 패턴 B — LucideIcon 타입 불일치 (12개 에러, 2개 파일)

**원인**: `React.ComponentType<{ size?: number; className?: string }>` 로 타입 지정 시
Lucide 아이콘의 `LucideProps.size`가 `string | number | null | undefined`라 `propTypes` 불일치 발생

**수정**: `type LucideIcon` import 후 해당 타입으로 교체

| 파일 | 수정 내용 |
|---|---|
| `components/board/TaskCard.tsx` | `Record<TaskEngine, React.ComponentType<...>>` → `Record<TaskEngine, LucideIcon>` |
| `pages/ResearchHub.tsx` | `Record<SourceType, React.ComponentType<...>>` → `Record<SourceType, LucideIcon>` |

### 패턴 C — `currentMissionId` 없는 속성 (3개 파일)

**원인**: `useAppStore`의 `AppState`에는 `currentMission: Mission | null`만 존재. `currentMissionId`는 없음

**수정 패턴**:
```typescript
// 수정 전
const { currentMissionId } = useAppStore()

// 수정 후
const { currentMission } = useAppStore()
const currentMissionId = currentMission?.id
```

| 파일 | 비고 |
|---|---|
| `pages/GrowthStudio.tsx` | `currentMissionId`를 하위 탭 컴포넌트에 prop으로 전달 |
| `pages/CDPView.tsx` | `identityApi.listProfiles()` queryKey/queryFn에 사용 |
| `pages/OpsCenter.tsx` | fetch body `mission_id`, `useCallback` deps에 사용 |

### 패턴 D — React Query v5 `onSuccess` 제거 (1개 파일)

**원인**: TanStack Query v5에서 `useQuery`의 `onSuccess` 콜백 옵션 삭제됨

**수정**: `pages/MissionBoard.tsx`

```typescript
// 수정 전
const { isLoading } = useQuery({
  ...
  onSuccess: (data) => setTasks(data),
})

// 수정 후
const { isLoading, data: queryData } = useQuery({ ... })

useEffect(() => {
  if (queryData) setTasks(queryData)
}, [queryData, setTasks])
```

### 패턴 F — SQLite boolean 비교 오류 (1개 파일)

**원인**: SQLite는 boolean을 `0/1`로 저장. `=== true` 비교는 `number`와 `boolean` 타입 간 비교로 TS 에러

**수정**: `pages/ResearchHub.tsx` — `=== true` 비교 3곳 제거

```typescript
// 수정 전
src.is_active === 1 || src.is_active === true

// 수정 후
src.is_active === 1
```

## 재발 방지 규칙

1. **Lucide 아이콘 타입**: `React.ComponentType<...>` 대신 `import { type LucideIcon } from 'lucide-react'` 사용
2. **React import**: `jsx: react-jsx` 프로젝트에서 `import React` 불필요 — 훅/타입만 named import
3. **AppStore 속성**: `currentMissionId` 없음. 항상 `currentMission?.id` 로 접근
4. **React Query v5**: `useQuery`에 `onSuccess` 없음. `useEffect`로 data sync
5. **SQLite boolean**: `is_active`, `is_custom` 등은 `0/1` 정수. `=== true` 비교 금지
6. **roleExecutors 경로**: `src/services/roleExecutors/` 하위에서 `src/` 루트 파일은 `../../` prefix 필요
