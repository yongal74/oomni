## v3.3.2 — Pencil 기술부채 완전 제거

### 배경

v3.3.0~v3.3.1에서 Pencil 관련 코드를 일부 제거했으나,
`claudeCodeService.ts`와 `XTerminal.tsx`에 Pencil 잔재가 남아 있어:
- 터미널에 "💡 Pencil MCP: Pencil 앱을 먼저 열고 /mcp 명령으로 연결하세요" 메시지 출력
- `designPencil` 시스템 프롬프트가 execute()에서 여전히 참조됨

또한 `claude-opus-4-7`가 실제로 존재하는 모델임을 확인 후 v3.3.1의 `claude-opus-4-6`에서 원복.

### 변경 파일

#### `frontend/src/components/bot/XTerminal.tsx`
```
제거: design role 연결 시 Pencil MCP 안내 메시지
      '\x1b[90m💡 Pencil MCP: Pencil 앱을 먼저 열고 /mcp 명령으로 연결하세요\x1b[0m'
제거: 주석 "design role: 백엔드가 Claude Code CLI + Pencil MCP로 실행"
```

#### `backend/src/services/claudeCodeService.ts`
```
제거: getRoleMcpConfig() 내 role === 'design' 블록 전체
      (Windows/Mac Pencil 바이너리 경로 탐색, pencil MCP 서버 설정)

제거: ROLE_PROMPTS.designPencil 시스템 프롬프트 전체
      (mcp__pencil__open_document, batch_design, get_screenshot 등 Pencil 명령 포함)

수정: execute() 메서드의 design role 분기 제거
      변경 전: this.role === 'design' ? ROLE_PROMPTS['designPencil'] ?? ... : ROLE_PROMPTS[this.role]
      변경 후: ROLE_PROMPTS[this.role] ?? ''
```

#### `backend/src/services/roleExecutors/design.ts`
```
수정: claude-opus-4-6 → claude-opus-4-7 (user confirmed 모델 존재)
```

## v3.3.1 — Design Bot 라우팅 근본 원인 수정 + Ops 인프라 스킬

### Design Bot 모델이 Sonnet 4.6으로 나오는 근본 원인

**문제**: `POST /api/agents/:id/chat` 엔드포인트에서 `design` 역할이
`claudeCodeRoles` 배열에 포함되어 있어 → `ClaudeCodeService` (Claude Code CLI)로 라우팅됨.
Claude Code CLI는 자체 설정 모델(Sonnet 4.6)을 사용 → `designExecutor`가 아예 호출되지 않음.

**수정** (`backend/src/api/routes/agents.ts`):
```typescript
// 변경 전
const claudeCodeRoles = ['design', 'build', 'project_setup', ...]

// 변경 후
const claudeCodeRoles = ['build', 'project_setup', ...]
// design 제거 → routeToExecutor → designExecutor → Anthropic SDK claude-opus-4-7
```

**Design 실행 경로 변경**:
```
변경 전: /chat → ClaudeCodeService → Claude Code CLI → Sonnet 4.6 → 터미널 JSON 출력
변경 후: /chat → routeToExecutor → designExecutor → Anthropic SDK → claude-opus-4-7 → HTML 스트리밍
```

**Design System 토큰 주입** (기존 `/stream` 엔드포인트와 동일하게):
- `design_systems` DB에서 `primary_color`, `bg_color`, `surface_color`, `text_color`, `font_family`, `border_radius` 조회
- task에 `[디자인 시스템 토큰]` 섹션으로 주입 후 `routeToExecutor` 호출

**`ctx.overrideModel` 강제 설정**:
```typescript
// design.ts - 모델 스위처 설정과 무관하게 항상 Opus 4.7 사용
ctx.overrideModel = 'claude-opus-4-7'
const result = await streamClaude(ctx, SYSTEM_PROMPT, task, 'claude-opus-4-7', 16000)
```
이유: `streamClaude` 내부에서 `ctx.overrideModel ?? model` 순서로 적용 →
프론트엔드 모델 스위처가 Sonnet으로 설정되어 있어도 Opus 4.7 강제 사용.

## v3.3.0 — Design Bot Pencil → HTML 스트리밍 교체

### 배경

Pencil MCP는 자동 기동/연결은 성공하나 OOMNI 중앙 패널 안에 디자인 캔버스가 표시되지 않음.
- Pencil: 독립 앱, iframe 내장 불가
- PNG export → 3초 폴링 방식은 UX 열악

Claude Design (Anthropic Research Preview) 분석 → `claude-opus-4-7`이 Claude Design 수준 HTML 생성 가능.
기존 `DesignCenterPanel`의 `extractHtml()` + `<iframe srcdoc>` 구조 재활용.

### 변경 사항

#### `backend/src/services/roleExecutors/design.ts`
- 모델: `undefined` (DEFAULT_MODEL = Sonnet 4.6) → `claude-opus-4-7`
- 기존 HTML 생성 시스템 프롬프트 유지 (300줄 이상, Tailwind CDN, 한국어 콘텐츠)

#### `frontend/src/components/bot/panels/DesignPanel.tsx`
- `PreviewTab` 타입: `'pencil' | 'html' | 'code'` → `'html' | 'code'`
- 기본 탭: `'pencil'` → `'html'`
- 제거: Pencil PNG 3초 폴링 `setInterval`, `imageTs`, `imageError` 상태
- 제거: "Pencil 미리보기" 탭 버튼 + PNG `<img>` 렌더링 블록
- 유지: `extractHtml()` + `<iframe srcDoc>` 실시간 스트리밍 렌더링

#### `backend/src/services/ptyService.ts`
- 제거: `design` role 전용 Pencil.exe 자동 기동 로직
  ```powershell
  # 제거됨
  $p = Get-Process "Pencil" -ErrorAction SilentlyContinue
  if (-not $p) { Start-Process "...\Pencil.exe"; Start-Sleep -Seconds 4 }
  ```
- 모든 역할 동일하게 `node "cliPath" --dangerously-skip-permissions` 실행

#### `backend/src/api/routes/agents.ts`
- 제거: `GET /api/agents/:id/preview-image` (Pencil PNG 서빙)
- 제거: `GET /api/agents/:id/pencil-status` (Pencil MCP 바이너리 확인)
- 제거: `import * as os from 'os'` (위 엔드포인트에서만 사용)
