# /investor-update — 투자자/멘토 업데이트 레터 자동 작성

월간 또는 분기별 투자자/멘토 업데이트 레터를 데이터 기반으로 작성한다. 수익 지표, 제품 진행, 팀 현황, 도움 요청사항까지 포함한 투명하고 신뢰감 있는 업데이트를 생성한다.

## 실행 단계

1. **데이터 수집**
   다음 파일 읽기:
   - `C:/oomni-data/ops/finance/monthly-finance_YYYY-MM.json` (최신 2개월)
   - `C:/oomni-data/growth/weekly-report_YYYY-WW.json` (최신)
   - `C:/oomni-data/ceo/okr_YYYY-QN.json` (현재 분기)
   - `C:/oomni-data/ceo/weekly-brief_YYYY-WW.json` (최신 4주)

2. **업데이트 레터 구조 설계**
   Paul Graham / Y Combinator 스타일 투자자 업데이트:
   ```
   1. 핵심 지표 요약 (3줄)
   2. 이번 달 달성한 것
   3. 이번 달 달성 못한 것 + 이유
   4. 다음 달 집중 계획
   5. 도움 요청 사항
   6. 상세 지표 (별첨)
   ```

3. **수익 섹션 작성**
   - MRR 현황 및 성장률
   - MRR 워터폴 (신규/확장/취소)
   - ARR 환산
   - 주요 고객 동향 (익명화)

4. **제품 섹션 작성**
   - 이번 달 출시한 기능 3개 이내
   - 주요 기술 부채 해결 사항
   - 다음 달 로드맵

5. **트랙션 섹션 작성**
   - 사용자 성장 (MoM)
   - 리텐션 트렌드
   - NPS (측정된 경우)

6. **솔직한 어려움 섹션 작성**
   투명성이 신뢰를 만든다. 솔직하게:
   - 예상보다 어려웠던 것
   - 실패한 실험
   - 해결 중인 문제

7. **도움 요청 작성**
   구체적인 요청 1-3가지:
   - 소개 요청 (누구를, 왜)
   - 전문가 의견 필요 사항
   - 리소스 요청

8. **레터 파일 저장**
   - 마크다운 버전 (내부 기록)
   - HTML 이메일 버전 (발송용)

## 출력 형식

MRR ₩2,480,000 (+12.2% MoM). D7 리텐션 41%로 개선 중. 결제 시스템 런치 완료.

## 핵심 지표

---

## 이번 달 달성한 것 ✅

1. **결제 시스템 런치**: Toss Payments + Stripe 동시 구현. 국내외 고객 모두 결제 가능.
2. **콘텐츠 엔진 가동**: 블로그 2편, 트위터 스레드 8개, 뉴스레터 4호 발행. 트위터 팔로워 +340명.
3. **AI API 비용 30% 절감**: claude-haiku 병행 사용으로 API 비용 ₩156,000 → ₩112,000.

## 달성 못한 것 + 이유 ❌

1. **온보딩 완료율 70% 목표 (현재: 55%)**: 온보딩 단계가 너무 복잡함. 다음 달 3단계로 축소 예정.
2. **Product Hunt 런치**: 기술적 준비는 됐지만 마케팅 준비가 부족해 다음 달로 연기.
3. **NPS 측정 시작 미완**: 인앱 서베이 도구 도입 중.

---

## 배운 것 💡

- **모바일 사용자 급증**: 전체 트래픽의 61%가 모바일. 모바일 최적화가 예상보다 훨씬 중요함.
- **트위터 > 블로그 (단기)**: 트위터 스레드 1개가 블로그 포스트 1개보다 10배 빠른 트래픽 유입.
- **무료 사용자 업셀**: 무료 플랜 한도 90% 도달 시 이메일 전환율 23% (기존 이메일 대비 4배).

---

## 다음 달 집중 계획

1. **온보딩 7→3단계 축소** (예상 D7 리텐션 +8%p)
2. **Product Hunt 런치** (4주 준비 후 5월 둘째 주 예정)
3. **이탈률 4% 이하** (At-Risk 이메일 캠페인 + 이탈 인터뷰)

1. **SaaS 온보딩 전문가 소개**: 온보딩 최적화 경험 있는 분을 30분만 만나고 싶습니다. 국내외 무관.

## 추가 인자

`$ARGUMENTS` — 선택적으로 다음을 지정할 수 있다:
- `--month YYYY-MM` : 특정 월 (기본값: 지난 달)
- `--recipient "투자자 이름"` : 수신자 이름 개인화
- `--type monthly|quarterly` : 업데이트 주기
- `--tone formal|casual` : 레터 톤 (기본값: casual)
- `--export html` : HTML 이메일 버전 생성
- 예시: `/investor-update --month 2026-03 --recipient "김민준 멘토님" --type monthly`
