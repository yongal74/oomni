# /thread-twitter — 트위터/X 바이럴 스레드 완전 작성

하나의 아이디어, 인사이트, 또는 스토리를 트위터/X 알고리즘과 독자 심리에 최적화된 스레드로 변환한다. 최대 도달률을 위한 구조, 포맷, 해시태그까지 포함한다.

## 실행 단계

1. **스레드 기획**
   - `$ARGUMENTS`에서 주제, 핵심 메시지, 타겟 독자 파싱
   - 스레드 타입 결정:
     - `story`: 개인 경험 스토리텔링
     - `lesson`: 배운 교훈 N가지
     - `how-to`: 단계별 방법 안내
     - `data`: 데이터와 인사이트
     - `contrarian`: 일반 통념에 반하는 주장
   - 최적 스레드 길이: 8-12개 트윗

2. **트윗 1: 훅 (가장 중요)**
   요소:
   - 첫 줄에서 스크롤 멈추게 하기
   - 숫자 포함 시 효과적 (예: "X년 동안 배운 Y가지")
   - "스레드를 읽을 이유" 명확히 제시
   - 280자 이내, 마지막에 "🧵" 이모지

   훅 패턴 5가지 생성:
   - 성과형: "[결과] 달성하는 데 [기간] 걸렸습니다. 비결은요:"
   - 의외성형: "모두가 [A]라고 하지만, 저는 [B]로 성공했습니다"
   - 실패형: "[실패 경험] 끝에 배운 [교훈]"
   - 질문형: "[공감 가는 질문]? 제가 알아봤습니다"
   - 숫자형: "X가지 [주제]를 [기간] 동안 해봤더니:"

3. **트윗 2-N: 본론**
   각 트윗:
   - 하나의 핵심 포인트만
   - 명확한 번호 매기기 (1/, 2/)
   - 들여쓰기/이모지로 시각적 구분
   - 각 트윗에서 다음 트윗을 보고 싶게 만드는 클리프행어

4. **중간 앵커 트윗**
   3-4번째 쯤에 스레드를 재RT 유도하는 트윗:
   "이 스레드 도움됐다면 리트윗으로 더 많은 사람들과 공유해주세요 ↑ 계속 이어집니다..."

5. **마지막 트윗: 요약 + CTA**
   - 핵심 포인트 불릿으로 요약
   - 팔로우 유도 (새 스레드 알림)
   - 링크 (블로그, 제품, 뉴스레터)
   - 저장/공유 부탁

6. **인게이지먼트 후속 트윗**
   스레드 발행 1시간 후 달 댓글용 트윗 초안:
   "가장 많이 물어본 질문에 답변드립니다..."

7. **발행 최적 시간 계산**
   한국 기준 최적 시간:
   - 평일: 오전 8-9시, 점심 12-1시, 저녁 9-11시
   - 주말: 오전 10-12시, 저녁 8-10시

## 추가 인자

`$ARGUMENTS` — 다음 형식으로 입력:
- `[주제] [핵심 메시지]`
- `--type story|lesson|how-to|data|contrarian` : 스레드 타입
- `--length 5|8|12|15` : 트윗 개수 (기본값: 10)
- `--hooks 3|5` : 훅 후보 개수
- `--source [파일경로]` : 원본 블로그 포스트 파일에서 자동 변환
- `--cta newsletter|product|blog` : CTA 대상
- 예시: `/thread-twitter "AI 도구 47개 실험" --type lesson --length 12 --cta newsletter`
